# 代码评审 (Code Review)

对未提交的更改进行全面的安全和质量评审：

1. 获取更改的文件：`git diff --name-only HEAD`

2. 针对每个更改的文件，检查：

**安全问题 (严重 - CRITICAL):**
- 硬编码的凭据、API 密钥、令牌
- SQL 注入漏洞
- XSS 漏洞
- 缺失输入验证
- 不安全的依赖项
- 路径遍历风险

**代码质量 (高 - HIGH):**
- 函数超过 50 行
- 文件超过 800 行
- 嵌套深度超过 4 层
- 缺失错误处理
- 存在 console.log 语句
- 存在 TODO/FIXME 注释
- 公共 API 缺失 JSDoc

**最佳实践 (中 - MEDIUM):**
- 变异模式 (建议使用不可变模式代替)
- 代码/注释中使用表情符号
- 新代码缺失测试
- 无障碍 (a11y) 问题

3. 生成报告，包含：
   - 严重程度：严重 (CRITICAL)、高 (HIGH)、中 (MEDIUM)、低 (LOW)
   - 文件位置和行号
   - 问题描述
   - 建议的修复方案

4. 如果发现“严重”或“高”优先级问题，则阻断提交。

严禁批准带有安全漏洞的代码！
