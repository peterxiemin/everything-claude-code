# /learn - 提取可复用模式

分析当前会话并提取任何值得保存为“技能”(skills) 的模式。

## 触发时机

在会话过程中解决了非琐碎问题后的任何时刻运行 `/learn`。

## 提取内容

关注以下内容：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 如何修复的？
   - 是否可复用于类似的错误？

2. **调试技巧**
   - 非显而易见的调试步骤
   - 行之有效的工具组合
   - 诊断模式

3. **变通方法 (Workarounds)**
   - 库的奇特行为
   - API 限制
   - 特定版本的修复

4. **项目特定模式**
   - 发现的代码库约定
   - 架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[pattern-name].md` 创建一个技能文件。

## 流程

1. 评审会话中可提取的模式
2. 识别最有价值/可复用的见解
3. 起草技能文件
4. 在保存前请求用户确认
5. 保存到 `~/.claude/skills/learned/`
