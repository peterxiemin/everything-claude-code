# 验证命令 (Verification Command)

对当前代码库状态运行全面的验证。

## 指令

按以下确切顺序执行验证：

1. **构建检查 (Build Check)**
   - 运行项目的构建命令
   - 如果失败，报告错误并停止

2. **类型检查 (Type Check)**
   - 运行 TypeScript/类型检查器
   - 报告所有带有 文件:行号 的错误

3. **代码规范检查 (Lint Check)**
   - 运行 Linter
   - 报告警告和错误

4. **测试套件 (Test Suite)**
   - 运行所有测试
   - 报告通过/失败的数量
   - 报告覆盖率百分比

5. **console.log 审计**
   - 在源文件中搜索 `console.log`
   - 报告位置

6. **Git 状态**
   - 显示未提交的更改
   - 显示自上次提交以来修改的文件

## 输出报告

生成一份简明扼要的验证报告：

```
验证结果：[通过/失败]

构建：    [正常/失败]
类型：    [正常/X 个错误]
规范：    [正常/X 个问题]
测试：    [X/Y 通过, Z% 覆盖率]
机密：    [正常/发现 X 个]
日志：    [正常/X 个 console.log]

准备好提交 PR：[是/否]
```

## 参数说明

$ARGUMENTS 可以是：
- `quick` - 仅执行构建 + 类型检查
- `full` - 执行所有检查 (默认)
- `pre-commit` - 提交前相关的检查
- `pre-pr` - 完整检查加安全扫描
