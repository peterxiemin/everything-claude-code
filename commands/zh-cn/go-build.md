# Go 构建与修复 (Go Build and Fix)

此命令调用 **go-build-resolver** 代理，以最小的变动增量修复 Go 构建错误。

## 此命令的作用

1. **运行诊断**：执行 `go build`, `go vet`, `staticcheck`
2. **解析错误**：按文件分组并按严重程度排序
3. **增量修复**：一次只修复一个错误
4. **验证每次修复**：每次更改后重新运行构建
5. **报告总结**：显示已修复的内容和剩余的问题

## 何时使用

在以下情况使用 `/go-build`：
- `go build ./...` 失败并报错
- `go vet ./...` 报告了问题
- `golangci-lint run` 显示警告
- 模块依赖关系损坏
- 拉取了破坏构建的更改后

## 修复策略

1. **构建错误优先** - 代码必须能够编译
2. **Vet 警告第二** - 修复可疑的结构
3. **Lint 警告第三** - 风格和最佳实践
4. **一次一个修复** - 验证每次更改
5. **最小化更改** - 不要重构，只修复错误

## 相关命令

- `/go-test` - 构建成功后运行测试
- `/go-review` - 评审代码质量
- `/verify` - 完整的验证循环
